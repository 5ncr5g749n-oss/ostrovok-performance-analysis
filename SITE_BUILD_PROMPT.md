### Секция 5: Стратегия (id="strategy")

Четыре Q&A блока. Каждый: вопрос из ТЗ жирным, ответ под ним.

**Q1: Каких данных критически не хватает?**
5 карточек в сетке 2×3:
1. 💰 Стоимость трафика (spend) — главный gap. Без него невозможно считать ROI.
2. 🔍 Глубина взаимодействия — время на сайте, просмотры без конверсии
3. 🗺 Параметры поиска — destination, даты, кол-во гостей
4. 📅 История до периода — новый ли пользователь в январе или returning
5. 🏨 Тип бронирования — сегмент отеля, направление

Блок внизу "Как действовать без данных":
- Использовать RPS как прокси доходности
- Ранжировать по CR + repeat rate
- Запросить spend из рекламных кабинетов

4 метода оценки верхнеуровневых каналов в виде пронумерованного списка карточек:

3. View-through attribution — конверсии от показов (не кликов)
4. Data-Driven attribution — при 10k+ конверсий/мес платформы дают модель

## Важные требования

1. **Все данные hardcoded** в JS внутри HTML — никаких fetch/ajax запросов
2. **Plotly через CDN** — без локальных файлов
3. **Один файл** — открывается в браузере без сервера
4. **Консистентная цветовая палитра каналов** во всех графиках
5. **Responsive** — на мобильном читаемо (KPI карточки в 2 колонки, таблицы со скроллом)
6. **Plotly config**: `{responsive: true, displayModeBar: false}` — без лишних кнопок на графиках
7. **Smooth scroll** при клике на навигацию
8. **Accordion** для субканалов — чистый JS, без библиотек
